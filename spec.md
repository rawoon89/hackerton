# spec.md — 멀티 리서처 BI

## 앱 한 줄 요약
자연어 질문 하나로 4명의 AI 리서처가 보험 데이터를 병렬 탐색하고, 편집자가 통합 인사이트 + 자동 차트를 제공하는 BI 웹앱

## 타겟 사용자
보험사 내부 기획/분석 담당자 — SQL 없이 자연어로 데이터 인사이트를 얻고 싶은 사람

## 핵심 기능
1. 자연어 질문 입력 + 샘플 질문 5개 원클릭
2. 4개 리서처 카드가 라이브로 탐색 상태를 표시
3. 편집자의 통합 답변 + 자동 선택 차트(bar/line/pie) + 인용 데이터 행
4. 관련 없는 리서처는 "무응답" 표기 (스마트 라우팅)
5. 거절 케이스 처리 (환각 없이 정중한 거절)

## 5 에이전트 역할 분담

### Agent 1: Router (쿼리 플래너)
- **역할**: 사용자 질문 분석 → 쿼리 플랜 JSON 생성
- **입력**: 사용자 자연어 질문
- **출력**: `{ researchers: ["product", "policy", ...], intent: "...", keywords: [...] }`
- **로직**: 질문 키워드 기반으로 관련 리서처만 활성화, 거절 케이스 판별

### Agent 2: Product Researcher
- **역할**: 보험 상품 카탈로그 탐색
- **데이터**: `products_catalog.csv`
- **출력**: `{ found: true, summary: "...", data_rows: [...], chart_suggestion: "bar" }`

### Agent 3: Policy Researcher
- **역할**: 고객 프로필 + 계약 + 보장 내역 교차 분석
- **데이터**: `customer_profiles.csv`, `policy_headers.csv`, `policy_coverages.csv`
- **출력**: 동일 JSON 스키마

### Agent 4: Loss Ratio Researcher
- **역할**: 손해율 시계열 분석
- **데이터**: `loss_ratio_timeseries.csv`
- **출력**: 동일 JSON 스키마

### Agent 5: Investment Researcher
- **역할**: 투자 상품/수익률/고객 포트폴리오 분석
- **데이터**: `investment_products.csv`, `customer_holdings.csv`, `nav_timeseries.csv`, `risk_profiles.csv`, `market_benchmarks.csv`, `transactions.csv`
- **출력**: 동일 JSON 스키마

> **편집자(Editor)는 별도 에이전트가 아닌 최종 합성 단계로 처리**
> Router가 라우팅 + 편집 역할을 겸함 (2-pass: 1차 라우팅 → 리서처 실행 → 2차 종합)

## 데이터 플로우

```
사용자 질문
    ↓
[Router Agent] → 쿼리 플랜 JSON (어떤 리서처를 활성화할지)
    ↓
[활성화된 리서처들] ← 각자 ./data/ CSV를 JS로 파싱+필터링
    ↓                   → 필터링된 데이터 + 질문을 Gemini에 전달
    ↓                   → 구조화된 JSON 응답 반환
    ↓
[Editor 단계] ← 모든 리서처 응답 수집
    ↓         → Gemini가 통합 답변 + 차트 타입 + 인용 생성
    ↓
UI 렌더링: 리서처 카드 4개 + 통합 답변 + 차트
```

## 기술 스택
- **프레임워크**: Next.js (App Router)
- **UI**: Tailwind CSS + shadcn/ui
- **차트**: Recharts
- **데이터 파싱**: Papa Parse
- **LLM**: Google Gemini API (Vercel AI SDK 경유)
- **배포**: Vercel
- **아이콘**: Lucide React
