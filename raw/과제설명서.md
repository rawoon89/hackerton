# A02 · 멀티 리서처 BI
> 자연어 질문 하나에 4명의 리서처 에이전트가 각자 다른 데이터셋을 병렬 탐색하고, 편집자 에이전트가 통합 답변을 돌려주는 자연어 BI 웹앱을 만들어 보세요.

먼저 [해커톤 심화 트랙 규칙](../해커톤_규칙_심화.md) 을 한 번 읽고 시작하세요.

**트랙:** 심화 · **예상 시간:** 2시간 · **산출물:** HTML + Serverless API (Gemini)

## 🎯 뭘 만드는 건가요
- **상단:** 자연어 질문 입력창 + 샘플 질문 5개 원클릭
- **중앙:** 4개 리서처 카드가 **라이브로 자기 데이터셋을 탐색**해요
  - `product_researcher` → `products_catalog`
  - `policy_researcher` → `customer_profiles` + `policy_headers` + `policy_coverages`
  - `loss_ratio_researcher` → `loss_ratio_timeseries`
  - `investment_researcher` → `set_g_investments` 6개 CSV
- **하단:** `editor` 에이전트의 **통합 답변** + 자동 선택된 차트 1개 + 인용된 데이터 행

## 🧱 이런 게 들어가면 좋아요
- 질문에 따른 **리서처 라우팅** (관련 없는 리서처는 "무응답" 표기)
- 리서처 응답 JSON 스키마 고정
- "답할 수 없음" 거절 케이스 처리
- 차트 타입 자동 선택 (bar / line / pie / none)

## 📂 받으시는 데이터 (`./data/`)
v2 세트 전체 플랫 복사:
- `products_catalog.csv`
- `customer_profiles.csv` / `policy_headers.csv` / `policy_coverages.csv`
- `loss_ratio_timeseries.csv`
- `investment_products.csv` / `customer_holdings.csv` / `nav_timeseries.csv` / `risk_profiles.csv` / `market_benchmarks.csv` / `transactions.csv`

## 🛠 시작하기
1. 이 폴더를 작업 위치로 복사하세요
2. `.env` 에 본인 `GEMINI_API_KEY` 를 설정하세요 ([Google AI Studio](https://aistudio.google.com/) 에서 발급)
3. 나머지는 여러분 몫이에요

## 🧠 고민해 볼 포인트
- 자연어 → **쿼리 플랜 JSON** 변환 프롬프트 품질
- 4 리서처를 전부 돌릴지 / 라우팅으로 1~2 개만 활성화할지
- CSV 집계를 **LLM 에 맡길지 / JS 에서 실행할지**
- 편집자 프롬프트의 **인용 강제**
- 차트 타입 자동 선택 로직

## 📝 스스로 테스트해 볼 수 있어요
**샘플 질문**
1. "30대 남성 고객이 가장 많이 가입한 상품 카테고리는?"
2. "최근 12개월 손해율이 가장 악화된 보험사는?"
3. "적극형 고객의 평균 월 투자 가용금액은?"
4. "자녀 둘 이상 가구가 관심 있는 보장은?"
5. (거절 케이스) "내일 주가가 오를까?" → 환각 없이 정중한 거절

## ✅ 완성 기준 (스스로 체크)
- [ ] 샘플 질문 4개에서 정확한 수치 답변 + 차트가 나와요
- [ ] 거절 질문에서 환각 없이 거절돼요
- [ ] 리서처 카드 4개가 UI 에 모두 표시돼요 (라우팅 결과로 "무응답" 허용)
